# 2. Техническое задание

## Общие сведения
- **Название проекта:** АлИИна — AI-консультант мебельной студии «Встрой-Мебель.ру»
- **Версия документа:** 1.0
- **Дата:** 2026-03-04
- **Основание:** Документ идеи проекта (`1_idea.md`)

## Цели и задачи

### Главная цель
Разработать AI-консультанта «АлИИна» для сайта мебельной студии «Встрой-Мебель.ру», способного вести диалог с клиентами в текстовой и голосовой форме, квалифицировать клиента, рассчитывать ориентировочную стоимость мебели и записывать на бесплатный замер — строго по заданному алгоритму и на основе предоставленных баз данных.

### Задачи
1. Реализовать чат-виджет для сайта с поддержкой текстового и голосового ввода/вывода
2. Интегрировать LLM-модель с системным промтом, реализующим алгоритм консультанта
3. Реализовать распознавание речи (STT) для приёма голосовых сообщений клиента
4. Реализовать синтез речи (TTS) для голосовых ответов консультанта
5. Организовать базы данных (DB-COMPANY-INFO, DB-PRICE-CATEGORY, DB-MODEL-NUMBERS, DB-EXCEPTION) с возможностью быстрого обновления
6. Реализовать алгоритмы расчёта стоимости мебели по категориям, моделям и специальным формулам
7. Обеспечить сбор и хранение данных клиента во внутреннем JSON
8. Реализовать передачу собранных данных менеджеру

## Функциональные требования

### Модуль 1 — Чат-виджет (Frontend)
- [ ] Встраиваемый виджет чата для сайта vstroy-mebel.ru
- [ ] Текстовое поле ввода с отправкой сообщений
- [ ] Кнопка записи голосового сообщения (микрофон)
- [ ] Отображение текстовых ответов консультанта в окне чата
- [ ] Кнопка воспроизведения голосового ответа консультанта (TTS)
- [ ] Автоматическое воспроизведение голосового ответа при голосовом режиме общения
- [ ] Отображение статуса: «АлИИна печатает...», «АлИИна говорит...»
- [ ] Возможность отправки файлов/фото (проект клиента)
- [ ] Адаптивный дизайн (десктоп и мобильные устройства)
- [ ] Кнопка сворачивания/разворачивания виджета

### Модуль 2 — Распознавание речи (STT)
- [ ] Захват аудио с микрофона пользователя через браузер (Web Audio API / MediaRecorder)
- [ ] Передача аудио на сервер для распознавания
- [ ] Конвертация голосового сообщения в текст (STT-сервис)
- [ ] Поддержка русского языка
- [ ] Отображение распознанного текста в чате перед отправкой
- [ ] Обработка ошибок распознавания (шум, неразборчивая речь)

### Модуль 3 — Синтез речи (TTS)
- [ ] Генерация голосового ответа из текста ответа консультанта
- [ ] Воспроизведение аудио-ответа в браузере клиента
- [ ] Естественное женское звучание голоса (соответствует образу «АлИИна»)
- [ ] Поддержка русского языка
- [ ] Возможность отключить голосовой режим (только текст)
- [ ] Потоковое воспроизведение (streaming) для минимизации задержки

### Модуль 4 — AI-движок (Backend, LLM)
- [ ] Интеграция с LLM-моделью (OpenAI GPT / аналог)
- [ ] Системный промт с полным алгоритмом консультанта (из `1_idea.md`)
- [ ] Контекст беседы: хранение истории сообщений в рамках сессии
- [ ] Внутренний JSON данных клиента (Client_name, Mebel_name, Dlina, Gorod, Ylitsa, Vremya_client, Client_Project, Orientir_price)
- [ ] Запрет на отображение внутреннего JSON клиенту
- [ ] Строгое следование алгоритму:
  1. Представление (один раз)
  2. Знакомство (обязательное получение имени)
  3. Консультация / уточнение категории мебели
  4. Вопрос перед расчётом (модель с сайта или по размерам)
  5. Подтверждение перед расчётом
  6. Расчёт стоимости
  7. Предложение замера
  8. Сбор контактных данных
  9. Завершение диалога

### Модуль 5 — Расчёт стоимости
- [ ] Расчёт по категории: `Стоимость = Цена категории × Длина (м)` — на основе [DB-PRICE-CATEGORY]
- [ ] Расчёт по номеру модели: `Стоимость = Цена модели × Длина (м)` — на основе [DB-MODEL-NUMBERS]
- [ ] Конвертация размеров: см → м, мм → м
- [ ] Извлечение длины из формата (159×60×240 → 1.59 м)
- [ ] Алгоритм для гардеробных:
  - Форма П: `(глубина × 2) + ширина`
  - Форма Г: `глубина + ширина`
  - Прямая: `ширина`
- [ ] Алгоритм для угловых шкафов: `ширина№1 + ширина№2`
- [ ] Алгоритм для угловых шкафов-купе: `ширина№1 + ширина№2`
- [ ] Обработка ссылок: извлечение названия и номера модели из URL (например, `uglovoi-shkaf-s-zerkalom-23` → №23)
- [ ] Запрет на отображение формулы и цены за метр клиенту
- [ ] Формат ответа: «Ориентировочная стоимость — XXXXX рублей...» с дисклеймером

### Модуль 6 — Базы данных
- [ ] **DB-COMPANY-INFO** — информация о компании (текстовая база): история, специализация, материалы, фурнитура, замер, оплата, доставка, сроки, гарантии, скидки, адрес, график работы
- [ ] **DB-PRICE-CATEGORY** — цены по категориям мебели за метр (9 категорий)
- [ ] **DB-MODEL-NUMBERS** — цены по номерам моделей (109+ позиций по 5 подкатегориям: шкафы-купе, распашные, гардеробные, прихожие, рабочие зоны)
- [ ] **DB-EXCEPTION** — исключения: кухни, ванные комнаты, сетчатые гардеробные, железные гардеробные
- [ ] Интерфейс администратора для быстрого обновления баз данных (CRUD)
- [ ] Формат хранения: JSON-файлы или таблицы БД с возможностью редактирования без развёртывания

### Модуль 7 — Обработка особых сценариев
- [ ] Работа с проектом клиента: приём файла/фото → фиксированное сообщение → запрос времени и телефона → передача менеджеру → остановка диалога
- [ ] Вызов менеджера: запрос телефона и удобного времени → «Всю информацию передам менеджеру» → остановка диалога
- [ ] Вмешательство менеджера в диалог → немедленная остановка диалога
- [ ] Запись на замер: поочерёдный сбор — город → улица → телефон
- [ ] После расчёта: предложение бесплатного замера

### Модуль 8 — Передача данных менеджеру
- [ ] Формирование итогового JSON с данными клиента
- [ ] Отправка данных менеджеру (email / Telegram-бот / CRM-интеграция)
- [ ] Уведомление менеджера о новой заявке
- [ ] Хранение истории диалогов для анализа

## Нефункциональные требования

- **Производительность:**
  - Время ответа чат-бота: не более 3–5 секунд (текст), не более 5–7 секунд (голос с TTS)
  - Потоковая генерация текста (streaming) для мгновенного отображения
  - Одновременная поддержка не менее 50 активных сессий

- **Безопасность:**
  - HTTPS для всех соединений
  - API-ключи LLM и TTS/STT хранятся на сервере, не передаются клиенту
  - Внутренний JSON клиента никогда не отображается в чате
  - Защита от инъекций в промт (prompt injection)
  - Соответствие ФЗ-152 (персональные данные клиентов)

- **Масштабируемость:**
  - Модульная архитектура: возможность замены LLM-провайдера, TTS/STT-сервиса
  - Базы данных обновляются без перезапуска системы
  - Возможность подключения на несколько сайтов

- **Доступность:**
  - Работа 24/7 (облачный хостинг)
  - Поддержка браузеров: Chrome, Firefox, Safari, Edge (последние 2 версии)
  - Мобильные браузеры: iOS Safari, Android Chrome
  - Graceful degradation: если голос недоступен — работа в текстовом режиме

- **Язык:**
  - Интерфейс и диалог только на русском языке

## Ограничения и допущения

### Ограничения консультанта
- Не сообщает цену за 1 метр клиенту
- Не показывает формулу расчёта
- Не знает стоимости: полок, ящиков, фурнитуры, дополнительных элементов
- Не делает расчёт без подтверждения клиента, при наличии проекта или без размеров
- Отвечает только в рамках задач компании
- Не занимается: кухнями, ванными, сетчатыми гардеробными, железными гардеробными

### Технические допущения
- Клиент использует современный браузер с поддержкой WebRTC / MediaRecorder
- Для голосового ввода требуется разрешение на доступ к микрофону
